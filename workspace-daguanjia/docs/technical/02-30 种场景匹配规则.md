定义 30 种常见防水场景的匹配规则，实现从用户输入的病害信息到施工方案的智能映射。

## 5. 匹配算法逻辑

### 5.1 匹配流程

```
用户输入 (病害 + 位置 + 严重程度)
  ↓
提取特征向量
  ↓
查询规则库
  ↓
计算匹配度得分
  ↓
按得分排序
  ↓
输出 TOP3 推荐方案
```

### 5.2 匹配度计算公式

```
匹配度得分 = (病害匹配分 × 0.4) + (位置匹配分 × 0.4) + (严重程度匹配分 × 0.2)
```

### 5.3 优先级排序规则

| 排序因素 | 权重 | 说明 |
|---------|------|------|
| 病害类型匹配 | 40% | 核心匹配因素 |
| 施工位置匹配 | 40% | 核心匹配因素 |
| 严重程度匹配 | 20% | 调整因素 |
| 用户评价加分 | +5% | 历史好评方案 |
| 成本优化加分 | +3% | 性价比高的方案 |
| 工期优势加分 | +2% | 快速施工方案 |

### 5.4 算法伪代码

```python
def match_scenarios(user_input):
    """
    用户输入：{disease_type, location, severity}
    返回：TOP3 推荐方案列表
    """
    candidates = []
    
    for scenario in SCENARIO_DATABASE:
        # 计算各维度匹配分
        disease_score = calc_disease_match(user_input.disease_type, scenario.disease_type)
        location_score = calc_location_match(user_input.location, scenario.location)
        severity_score = calc_severity_match(user_input.severity, scenario.severity)
        
        # 加权计算总分
        total_score = (
            disease_score * 0.4 +
            location_score * 0.4 +
            severity_score * 0.2
        )
        
        # 加分项
        if scenario.user_rating >= 4.5:
            total_score += 0.05
        if scenario.cost_efficiency == 'high':
            total_score += 0.03
        if scenario.construction_days <= 3:
            total_score += 0.02
        
        candidates.append({
            'scenario': scenario,
            'score': total_score
        })
    
    # 排序并返回 TOP3
    candidates.sort(key=lambda x: x['score'], reverse=True)
    return candidates[:3]
```

## 6. TOP3 推荐方案筛选规则

### 6.1 筛选标准

| 排名 | 筛选标准 | 展示重点 |
|-----|---------|---------|
| **TOP1** | 匹配度最高 (≥85 分) | "最推荐"标签 + 核心优势 |
| **TOP2** | 匹配度次高 (70-84 分) | "性价比高"或"工期短"亮点 |
| **TOP3** | 匹配度第三 (60-69 分) | "备选方案"或"高端选择" |

| 情况 | 处理规则 |
|-----|---------|
| 匹配度<60 分 | 显示"建议预约顾问上门评估" |
