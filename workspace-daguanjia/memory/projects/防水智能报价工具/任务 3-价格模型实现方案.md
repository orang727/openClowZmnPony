## 一、价格模型设计

### 1.1 定价公式

```
最终价格 = 基础价格 × 位置系数 × 难度系数 × 渗漏系数 + 附加费用
```

### 1.2 参数说明

| 参数 | 说明 | 取值范围 | 来源 |
|------|------|---------|------|
| **基础价格** | 基于面积和位置的基准价格 | 500-8000 元 | 历史数据统计 |
| **位置系数** | 不同施工位置的难度差异 | 0.9-1.3 | 业务专家确认 |
| **难度系数** | 施工难度等级 | 0.8-1.8 | 业务专家确认 |
| **渗漏系数** | 渗漏严重程度 | 0.8-1.5 | 业务专家确认 |
| **附加费用** | 特殊情况的额外费用 | 0-2000 元 | 实际发生 |

### 1.3 系数表

#### 位置系数

| 位置 | 系数 | 说明 |
|------|------|------|
| 卫生间 | 1.0 | 标准施工（基准） |
| 厨房 | 0.9 | 施工难度略低 |
| 阳台 | 0.9 | 施工难度略低 |
| 屋顶 | 1.2 | 高空作业 |
| 外墙 | 1.3 | 高空作业 + 难度高 |
| 飘窗 | 1.1 | 难度中等 |
| 其他 | 1.0 | 默认 |

#### 难度系数

| 等级 | 系数 | 说明 | 判定标准 |
|------|------|------|---------|
| 轻微 | 0.8 | 小面积，简单施工 | 面积<5㎡，表面问题 |
| 普通 | 1.0 | 标准施工 | 面积 5-15㎡，常规问题 |
| 严重 | 1.5 | 大面积，复杂施工 | 面积>15㎡，结构问题 |

#### 渗漏系数

| 等级 | 系数 | 说明 | 判定标准 |
|------|------|------|---------|
| 轻微 | 0.8 | 表面渗漏 | 仅表面潮湿，无渗透 |
| 普通 | 1.0 | 轻微渗漏 | 有渗透，但未扩散 |
| 严重 | 1.5 | 严重渗漏 | 大面积渗透，影响结构 |

### 1.4 附加费用项

| 费用项 | 金额 | 触发条件 |
|--------|------|---------|
| 高空作业费 | 200-500 元 | 外墙/屋顶施工 |
| 夜间施工费 | 300 元 | 用户要求夜间施工 |
| 紧急加急费 | 500 元 | 24 小时内上门 |
| 家具保护费 | 100-300 元 | 需要保护贵重家具 |
| 垃圾清运费 | 200 元 | 需要清理建筑垃圾 |
| 特殊材料费 | 实报实销 | 用户指定特殊材料 |

### 3.3 价格计算伪代码

```javascript
// 价格计算核心逻辑
function calculatePrice(params) {
  // 1. 获取基础价格（从场景映射表）
  let basePrice = getBasePrice(params.sceneId);
  
  // 2. 获取系数
  let locationCoef = getLocationCoef(params.locationType);
  let difficultyCoef = getDifficultyCoef(params.severity);
  let leakageCoef = getLeakageCoef(params.leakageLevel);
  
  // 3. 计算核心价格
  let corePrice = basePrice * locationCoef * difficultyCoef * leakageCoef;
  
  // 4. 添加附加费用
  let additionalCost = 0;
  if (params.isHighAltitude) additionalCost += 300;
  if (params.isUrgent) additionalCost += 500;
  if (params.needFurnitureProtection) additionalCost += 200;
  
  // 5. 最终价格
  let finalPrice = corePrice + additionalCost;
  
  // 6. 返回价格区间（考虑误差）
  return {
    minPrice: Math.round(finalPrice * 0.8),  // -20%
    maxPrice: Math.round(finalPrice * 1.2),  // +20%
    recommendPrice: Math.round(finalPrice),
    breakdown: {
      basePrice: Math.round(basePrice),
      locationCoef: locationCoef,
      difficultyCoef: difficultyCoef,
      leakageCoef: leakageCoef,
      additionalCost: additionalCost
    }
  };
}
```

## 四、实现方案

### 4.1 前端计算 vs 后端计算

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **前端计算** | 响应快、节省服务器资源 | 逻辑暴露、易被篡改、难维护 | ⭐⭐ |
| **后端计算** | 逻辑安全、易维护、易更新 | 增加服务器压力、响应略慢 | ⭐⭐⭐⭐⭐ |

**推荐方案：后端计算**

**理由**：
1. 价格逻辑是核心商业机密，不宜暴露
2. 系数调整频繁，后端更新无需发版
3. 便于统一风控和日志记录
4. 响应时间差异可忽略（<100ms）

## 五、风控机制设计

### 5.1 价格异常检测

**检测规则**：

| 规则 | 阈值 | 处理方式 |
|------|------|---------|
| AI 估价与最终报价偏差 | >30% | 触发人工审核 |
| 单价异常 | <300 元/㎡ 或 >2000 元/㎡ | 触发预警 |
| 折扣异常 | <0.7 或 >1.0 | 触发审批 |
| 附加费用占比 | >30% | 触发审核 |
| 同一师傅连续低价 | 3 单以上 | 触发调查 |

### 5.2 风控流程

```
报价生成 → 规则检测 → 正常 → 显示给用户
              │
              ↓ 异常
         触发预警 → 人工审核 → 调整报价 → 显示给用户
```

### 5.3 黑白名单机制

**师傅黑名单**：
- 连续 3 次报价偏差>50%
- 用户投诉>3 次
- 发现恶意低价竞争

**处理方式**：
- 暂停派单
- 重新培训
- 严重者解除合同

### 5.4 日志记录

**记录内容**：
- 每次报价的输入参数
- 计算过程和中间结果
- 最终报价和修改记录
- 用户反馈和实际成交价

**保存期限**：3 年
