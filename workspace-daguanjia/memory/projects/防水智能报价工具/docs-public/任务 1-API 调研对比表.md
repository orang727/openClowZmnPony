## 一、调研目标

识别防水补漏场景中的：
1. **面积估算**：从图片中识别施工区域，估算面积（㎡）
2. **病害类型**：识别渗漏类型（裂缝/空鼓/霉变/渗水等）
3. **严重程度**：判断轻微/普通/严重等级

## 四、测试效果评估方案

### 4.1 准确率测试方案

**测试数据集**：
- 收集 100 张真实防水施工场景图片（涵盖不同位置、光线、角度）
- 人工标注：面积（长×宽）、病害类型、严重程度
- 分类：卫生间 30 张、厨房 20 张、阳台 20 张、屋顶 15 张、外墙 15 张

**测试指标**：
| 指标 | 目标值 | 计算方式 |
|------|--------|---------|
| 面积估算准确率 | >80% | |估算值 - 真实值| / 真实值 < 20% |
| 病害类型识别准确率 | >85% | 正确识别数 / 总数 |
| 严重程度判断准确率 | >80% | 正确判断数 / 总数 |
| 综合准确率 | >80% | 加权平均 |

**测试流程**：
1. 准备测试数据集（3/10 前完成）
2. 调用各 API 进行测试（3/11 前完成）
3. 统计准确率，输出测试报告（3/11 前完成）

### 4.2 响应时间测试

**测试方法**：
- 每 API 连续调用 100 次，记录响应时间
- 统计：平均响应时间、P95 响应时间、超时率

**目标**：
- 平均响应时间：<500ms
- P95 响应时间：<1000ms
- 超时率：<1%

### 4.3 稳定性评估

**测试方法**：
- 连续 7 天，每天调用 1000 次
- 记录：成功率、错误类型、服务中断次数

**目标**：
- 成功率：>99%
- 服务中断：0 次

## 五、推荐方案

### 5.1 分阶段推荐

| 阶段 | 时间 | 推荐方案 | 理由 |
|------|------|---------|------|
| **MVP 阶段** | 0-3 个月 | **百度 AI** | 成本最低、免费额度高、快速验证 |
| **增长阶段** | 3-12 个月 | **阿里云** | 稳定性好、生态完善、支持业务增长 |
| **成熟阶段** | 12 个月+ | **自建模型** | 数据积累充足后，自建降低成本、提升准确率 |

### 5.2 MVP 阶段详细建议

**推荐：百度 AI 开放平台**

**核心 API**：
1. **图像分割**：识别施工区域边界
2. **物体检测**：识别漏点位置（卫生间/厨房等）
3. **EasyDL 自定义模型**：训练防水病害识别模型（可选）

**成本估算**：
- 月调用量：10 万次
- 月度成本：约 600-900 元
- 免费额度：5000 次/月（图像识别）

**实施步骤**：
1. 注册百度 AI 账号，创建应用（3/10）
2. 获取 API Key 和 Secret Key（3/10）
3. 调用图像分割 API 测试（3/11）
4. 基于测试结果调整参数（3/11）
5. 集成到 H5 产品（3/15 后）

**风险应对**：
- 准确率低：增加用户手动修正入口（已在 PRD 中设计）
- 响应慢：设置超时降级（>3 秒切换手动模式）
- 服务不可用：备用阿里云 API（双供应商）
