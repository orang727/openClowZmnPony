## 异常流程设计方案

### 1. 网络超时处理

**触发条件**: 
- 图片上传时网络请求超过 15 秒无响应
- AI 分析请求超过 30 秒无响应
- 工单提交请求超过 10 秒无响应

**用户看到**: 
```
┌─────────────────────────────┐
│  ⚠️  网络连接超时          │
├─────────────────────────────┤
│                             │
│  网络好像有点慢             │
│  请检查网络设置后重试       │
│                             │
│  ┌─────────────────────┐   │
│  │      [ 重新尝试 ]   │   │
│  └─────────────────────┘   │
│                             │
│  [ 返回上一步 ]             │
│                             │
└─────────────────────────────┘
```

**系统处理**: 
1. 前端检测到请求超时（设定 timeout 阈值）
2. 显示友好错误提示，不暴露技术细节
3. 保留用户已输入的数据（本地缓存）
4. 记录错误日志（用于后续优化）

**兜底方案**: 
- 自动重试机制：最多自动重试 2 次（间隔 3 秒）
- 降级方案：如 AI 分析超时，提供"跳过 AI 分析，手动选择"选项
- 离线模式：允许用户先填写信息，网络恢复后自动提交

### 2. AI 分析失败处理

**触发条件**: 
- AI 服务返回错误码（5xx/4xx）
- 图片无法识别（模糊、过暗、无漏水特征）
- AI 服务不可用（服务宕机）

**用户看到**: 
```
┌─────────────────────────────┐
│  ⚠️  AI 分析未成功          │
├─────────────────────────────┤
│                             │
│  抱歉，AI 暂时无法识别       │
│  您的照片                   │
│                             │
│  可能原因：                 │
│  • 照片太暗或模糊           │
│  • 照片中没有明显漏水       │
│  • 拍摄角度不理想           │
│                             │
│  ┌─────────────────────┐   │
│  │  [ 重新上传照片 ]   │   │
│  └─────────────────────┘   │
│                             │
│  ┌─────────────────────┐   │
│  │  [ 手动选择方案 ]   │   │
│  └─────────────────────┘   │
│                             │
└─────────────────────────────┘
```

**系统处理**: 
1. 捕获 AI 服务错误，记录详细错误信息
2. 判断错误类型（图片质量问题 vs 服务问题）
3. 提供明确的错误原因和解决建议
4. 提供备选方案（手动选择/转人工）

**兜底方案**: 
- **方案 A**: 用户手动选择漏点位置和严重程度
- **方案 B**: 转人工客服（在线/电话）
- **方案 C**: 先提交工单，师傅上门后评估

### 3. 图片上传失败处理

**触发条件**: 
- 图片格式不支持（非 jpg/png/gif）
- 图片大小超限（>8MB）
- 上传过程中网络中断
- 服务器存储失败

**用户看到**: 
```
┌─────────────────────────────┐
│  ❌  上传失败               │
├─────────────────────────────┤
│                             │
│  [图片缩略图]               │
│  IMG_20260308.jpg           │
│                             │
│  失败原因：图片大小超限     │
│  (最大支持 8MB)             │
│                             │
│  ┌─────────────────────┐   │
│  │      [ 重新选择 ]   │   │
│  └─────────────────────┘   │
│                             │
│  已上传 2/5 张              │
│  [继续上传] [跳过]          │
│                             │
└─────────────────────────────┘
```

**系统处理**: 
1. 前端预检查（格式、大小）后再上传
2. 上传失败时显示具体原因
3. 保留已成功上传的图片
4. 支持单张重传，不影响其他图片

**兜底方案**: 
- 允许用户跳过图片上传（最低 1 张）
- 提供图片压缩建议（"您可以压缩图片后重试"）
- 支持从相册重新选择

### 4. 用户中途退出处理

**触发条件**: 
- 用户主动关闭页面/返回
- 浏览器崩溃/刷新
- 切换应用导致页面失效

**用户看到**: 
```
┌─────────────────────────────┐
│  💾  进度已保存             │
├─────────────────────────────┤
│                             │
│  欢迎回来！                 │
│  检测到您有未完成的报价     │
│                             │
│  进度：P3 - AI 分析中        │
│  时间：2026-03-08 12:30     │
│                             │
│  ┌─────────────────────┐   │
│  │  [ 继续报价 ]       │   │
│  └─────────────────────┘   │
│                             │
│  [ 重新开始 ]               │
│                             │
└─────────────────────────────┘
```

**系统处理**: 
1. 每步操作后自动保存进度到 localStorage
2. 关键数据同步到服务器（选择的位置、上传的图片）
3. 用户再次访问时检测本地缓存
4. 提供"继续"或"重新开始"选项

**兜底方案**: 
- 本地缓存有效期：7 天
- 超过有效期提示"进度已过期"
- 支持通过手机号恢复云端保存的进度

│                             │
│  ┌─────────────────────┐   │

## 异常处理原则

### 设计原则

1. **用户友好** - 不暴露技术细节，用用户语言描述
2. **明确指引** - 告诉用户发生了什么、如何解决
3. **保留数据** - 尽可能保存用户已输入的信息
4. **提供备选** - 每个异常都有兜底方案
5. **及时恢复** - 支持重试、继续、撤销

### 技术原则

1. **前端预检** - 格式、大小等在前端校验
2. **超时控制** - 所有请求设置合理 timeout
3. **错误捕获** - 完善的错误处理和日志
4. **降级方案** - 核心功能必须有备选方案
5. **监控告警** - 异常及时通知团队

## 异常状态码映射

| 场景 | 错误码 | 用户提示 | 处理方式 |
|------|--------|---------|---------|
| 网络超时 | TIMEOUT | 网络连接超时 | 重试/检查网络 |
| AI 分析失败 | AI_ERROR | AI 分析未成功 | 手动选择/转人工 |
| 图片上传失败 | UPLOAD_ERROR | 上传失败 | 重传/压缩 |
| 面积估算失败 | AREA_ERROR | 请手动输入面积 | 手动输入 |
| 价格计算失败 | PRICE_ERROR | 价格计算不可用 | 客服报价 |
| 工单推送失败 | PUSH_ERROR | 提交成功（后台处理） | 异步重试 |
