## 🎯 重要里程碑：用户测试

### 需求理解

**用户确认** = **用户测试**（不是页面功能）

**目的**:
- 找真实用户（师傅/业主）走完整流程
- 观察用户行为和操作意愿
- 统计各环节转化率
- 收集反馈优化产品
- **作为重要里程碑，算法成熟后执行**

---

## 📋 测试方案框架

### 测试目标

| 目标 | 说明 |
|------|------|
| 页面易用性 | 用户能否顺利完成流程 |
| 报价接受度 | 用户对报价的接受程度 |
| 留资意愿 | 用户是否愿意留下联系方式 |
| 下单意愿 | 用户是否愿意实际下单 |

### 转化漏斗指标

```
访问用户 (100%)
   ↓
提交信息用户 (？%)
   ↓
查看报价用户 (？%)
   ↓
留资用户 (？%)
   ↓
实际下单用户 (？%) ← 核心指标
```

### 测试样本

| 用户类型 | 建议样本量 | 说明 |
|----------|-----------|------|
| C 端业主 | 20-30 人 | 真实需求用户 |
| B 端师傅 | 10-15 人 | 勘察/施工方 |
| **合计** | **30-45 人** | 覆盖不同场景 |

### 测试场景

| 场景 | 样本数 | 说明 |
|------|--------|------|
| 卫生间渗漏 | 10-15 | 最常见场景 |
| 厨房渗漏 | 5-8 | 常见场景 |
| 阳台渗漏 | 5-8 | 常见场景 |
| 其他（屋顶/外墙） | 5-10 | 补充场景 |

### 测试方式

| 方式 | 优点 | 缺点 |
|------|------|------|
| 现场观察 | 可观察真实行为 | 成本高 |
| 远程录屏 | 成本低，可批量 | 缺少互动 |
| 访谈 + 测试 | 深度反馈 | 耗时 |

**建议**: 现场观察 (5 人) + 远程录屏 (25 人)

**均未达标** → 重新评估产品方向
