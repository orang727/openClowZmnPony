## 一、核心流程

```
用户提交 → AI 初步分析 → 师傅实地勘察 → 确认/修正 → 生成施工方案 → 报价 → 用户确认 → 施工
```

**关键变更**：
- AI 分析结果仅作为**初步参考**，不作为最终施工方案
- 师傅实地勘察是**必须环节**，确保方案准确性
- 用户确认后方可施工，避免纠纷

## 三、实地勘察流程设计

### 3.1 勘察检查清单

**师傅上门勘察必查项**：

| 检查项 | 检查内容 | 记录方式 |
|--------|---------|---------|
| **位置确认** | 确认漏点位置（与用户描述是否一致） | 拍照 + 勾选 |
| **面积测量** | 实际测量施工区域长×宽 | 卷尺测量 + 记录 |
| **病害类型** | 裂缝/空鼓/渗水/霉变 | 拍照 + 勾选 |
| **严重程度** | 轻微/普通/严重 | 拍照 + 勾选 |
| **周边环境** | 是否有家具/电器需要保护 | 拍照 + 记录 |
| **施工难度** | 是否需要高空作业/特殊设备 | 勾选 |
| **隐藏问题** | 墙体内部渗水/管道问题 | 专业设备检测 + 记录 |
| **用户特殊需求** | 工期要求/特殊材料偏好 | 文字记录 |

### 3.3 勘察流程

```
1. 预约上门（客服联系用户，确认时间）
   ↓
2. 师傅准时到达（签到打卡）
   ↓
3. 实地勘察（使用检查清单，逐项检查）
   ↓
4. 拍照记录（关键位置拍照上传系统）
   ↓
5. 生成方案（系统自动推荐 + 师傅调整）
   ↓
6. 报价确认（向用户展示最终报价）
   ↓
7. 用户签字（电子签名或纸质签名）
   ↓
8. 上传系统（勘察结果同步到工单系统）
   ↓
9. 安排施工（用户确认后安排施工时间）
```

## 四、数据模型方案

### 4.1 利用 25 年历史数据

**数据价值**：
- 积累真实施工案例
- 优化价格系数
- 提升 AI 识别准确率
- 预测施工周期和成本

**数据字段需求**（给泊宁导表）：

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| order_id | string | 工单 ID | ORD20250101001 |
| create_time | datetime | 创建时间 | 2025-01-01 10:00:00 |
| location_type | string | 漏点位置 | 卫生间 |
| area | number | 面积（㎡） | 6.5 |
| damage_type | string | 病害类型 | 裂缝 |
| severity | string | 严重程度 | 普通 |
| construction_plan | string | 施工方案 | 局部注浆 + 表面防水 |
| difficulty_coef | number | 难度系数 | 1.0 |
| leakage_coef | number | 渗漏系数 | 1.0 |
| material_cost | number | 材料费 | 400 |
| labor_cost | number | 人工费 | 600 |
| total_price | number | 总价 | 1000 |
| actual_price | number | 实际成交价 | 980 |
| construction_days | number | 施工天数 | 2 |
| warranty_years | number | 质保年限 | 3 |
| customer_satisfaction | number | 用户满意度 (1-5) | 4.8 |
| rework_flag | boolean | 是否返工 | false |
| master_id | string | 师傅 ID | M001 |
| master_experience | number | 师傅工龄（年） | 10 |

### 4.2 数据模型设计

**模型目标**：
1. **价格预测模型**：基于历史数据预测合理价格区间
2. **施工周期预测**：预测施工所需天数
3. **满意度预测**：预测用户满意度，提前识别风险

**建模方法**：
- 回归模型：预测价格、施工周期
- 分类模型：预测满意度等级
- 聚类分析：识别相似案例

**实施步骤**：
1. 数据清洗（3/15-3/20）
2. 特征工程（3/20-3/25）
3. 模型训练（3/25-4/1）
4. 模型验证（4/1-4/5）
5. 上线部署（4/5-4/10）
