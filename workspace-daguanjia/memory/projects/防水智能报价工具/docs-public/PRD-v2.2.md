## 📊 战略背景

### 市场机遇
- **市场规模**: 200-300 亿家庭防水市场
- **线上渗透**: 约 15%（30-45 亿）
- **用户需求**: 透明报价、快速响应、专业服务

### 行业痛点
| 痛点 | 用户视角 | 平台视角 |
|------|---------|---------|
| 报价不透明 | 不知道合理价格范围 | 获客成本高，转化率低 |
| 师傅随意报价 | 担心被坑，信任度低 | 服务质量难以标准化 |
| 决策周期长 | 多家对比，犹豫不决 | 线索流失率高 |

### 战略价值
- **获客突破**: 降低获客成本，提升转化率
- **品牌信任**: 透明报价建立用户信任
- **数据积累**: 积累报价数据，优化算法

## 🎯 三大差异化策略

### 策略 1：价格保障机制
**核心卖点**：透明报价，贵一赔十

| 保障措施 | 说明 |
|----------|------|
| **报价透明** | 明确显示价格构成（材料 + 人工 + 难度系数） |
| **误差保障** | AI 估价与最终报价误差±20% 内不补差价 |
| **贵一赔十** | 如发现高于市场均价 10%，差价 10 倍赔偿 |
| **官方质保** | 所有订单享受官方质保（1-5 年） |

### 策略 2：AI+ 人工双保险
**核心卖点**：智能估价，人工确认，双重准确

| 环节 | AI 职责 | 人工职责 |
|------|--------|---------|
| **面积估算** | AI 拍照估算（可选） | 用户手动输入修正 |
| **严重程度** | AI 图像识别 | 师傅上门复核 |
| **最终报价** | AI 初步估价 | 师傅确认后调整 |
| **施工方案** | AI 推荐方案 | 师傅优化执行 |

### 策略 3：闭环服务体验
**核心卖点**：从估价到完工，一站式搞定

| 阶段 | 服务内容 | 用户价值 |
|------|---------|---------|
| **估价阶段** | AI 智能报价 + 方案推荐 | 透明、快速 |
| **留资阶段** | 专属客服对接 + 优先派单 | 省心、高效 |
| **上门阶段** | 师傅复核 + 最终确认 | 准确、可靠 |
| **施工阶段** | 标准化施工 + 进度跟踪 | 专业、透明 |
| **售后阶段** | 官方质保 + 问题响应 | 安心、保障 |

## ⚠️ AI 估价局限性说明

### 必须向用户明确说明

**展示位置**：价格区间显示页面底部

**说明文案**：
```
⚠️ AI 估价说明

1. 本价格为 AI 基于您提供的图片和信息进行的估算，仅供参考
2. 最终价格以师傅上门测量后确认的报价为准
3. AI 估价可能存在±20% 的误差，误差范围内不补差价
4. 以下情况可能影响估价准确性：
   - 图片不清晰或光线不足
   - 拍摄角度导致面积判断偏差
   - 隐藏渗漏点未被发现
   - 特殊施工难度未识别
5. 师傅上门后如发现实际情况与图片差异较大，会及时与您沟通调整方案
```

### 技术局限性

| 局限 | 影响 | 应对方案 |
|------|------|---------|
| 单张照片无参照物 | 面积估算误差大（±30-50%） | 引导用户手动输入长宽 |
| 透视变形 | 形状识别不准确 | 多角度拍照 + AI 校正 |
| 光线不足 | 严重程度判断偏差 | 提示用户改善光线 |
| 隐藏渗漏 | 无法识别墙体内部 | 师傅上门用专业设备检测 |
| 不规则形状 | 面积计算复杂 | 提供手动修正入口 |

## 🔄 异常流程处理

### 异常场景及处理方案

| 异常场景 | 触发条件 | 处理方案 | 用户体验 |
|----------|---------|---------|---------|
| **图片上传失败** | 网络问题/文件过大 | 1. 提示重试 2. 支持压缩 3. 允许跳过 | 不影响流程 |
| **AI 识别失败** | API 超时/返回错误 | 1. 自动重试 2. 降级为手动输入 3. 提示用户 | 无缝切换 |
| **面积估算异常** | 用户输入不合理数值 | 1. 范围校验 2. 提示合理范围 3. 提供案例参考 | 引导修正 |
| **价格计算异常** | 系数缺失/算法错误 | 1. 使用默认值 2. 显示价格区间 3. 提示人工确认 | 保证可用 |
| **留资表单放弃** | 用户填写中途离开 | 1. 本地缓存 2. 短信提醒（如已填手机）3. 允许匿名查看估价 | 降低流失 |
| **工单推送失败** | 工单系统异常 | 1. 本地存储 2. 定时重试 3. 人工介入 | 保证不丢单 |
| **并发过高** | 流量激增 | 1. 限流降级 2. AI 服务降级 3. 排队提示 | 保证核心功能 |

### 降级策略

**Level 1（轻度降级）**：
- AI 识别超时→切换备用 API
- 图片上传慢→提示压缩

**Level 2（中度降级）**：
- AI 服务不可用→纯手动输入模式
- 报价算法异常→显示参考区间

**Level 3（重度降级）**：
- 系统不可用→展示客服联系方式
- 工单系统异常→人工派单
