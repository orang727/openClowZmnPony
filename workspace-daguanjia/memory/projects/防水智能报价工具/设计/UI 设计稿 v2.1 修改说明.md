## 🔴 P1 优先级修改（必须完成）

### 1. P3 默认状态调整

**修改内容**:
- 默认状态从"AI 拍照估算"改为"手动输入面积"
- AI 智能识别从默认选项改为可选按钮
- 添加提示文案"AI 识别仅供参考，建议手动输入确认"

**修改原因**:
- 降低用户对 AI 的过度依赖
- 提高面积数据准确性
- 管理用户预期，避免纠纷

**设计变更**:
```
v2.0: AI 拍照估算 [✓]  ← 默认选中
      手动输入 [ ]

v2.1: 手动输入 [✓]  ← 默认选中
      AI 拍照估算 [ ]
      
      💡 AI 识别仅供参考，建议手动输入确认
```

**影响范围**: P3 页面、交互逻辑

---

### 2. P5 表单简化

**修改内容**:
- 留资表单字段从 3 个简化为 2 个（姓名 + 电话）
- 地址字段移动到 P6 确认下单页
- 添加表单填写进度提示（步骤 1/2）

**修改原因**:
- 减少用户填写负担，提高转化率
- 地址信息在确认页填写更合理
- 进度提示增强用户掌控感

**设计变更**:
```
v2.0 P5 表单字段:
  - 姓名 *
  - 手机号 *
  - 小区/详细地址 *  ← 移至 P6
  - 期望上门时间

v2.1 P5 表单字段:
  - 姓名 *
  - 手机号 *
  - 💡 详细地址将在下一页填写
  
v2.1 P6 新增字段:
  - 上门服务地址 *  ← 从 P5 移入
```

**影响范围**: P5 页面、P6 页面、表单校验逻辑

---

### 3. 价格计算逻辑明确

**修改内容**:
- P5 显示"估算价格范围"（非最终价格）
- 添加说明"最终价格以上门评估为准"
- 增加"预约顾问获取精确报价"选项

**修改原因**:
- 避免价格纠纷，明确价格性质
- 提供人工服务选项，提高转化率
- 管理用户预期，减少投诉

**设计变更**:
```
v2.0:
      ¥2000 - ¥3500
      最终价格以师傅上门评估为准

v2.1:
      ¥2000 - ¥3500
      
      💡 估算价格范围
         最终价格以上门评估为准
      
      ┌─────────────────────┐
      │  📞 预约顾问        │
      │     获取精确报价 →   │
      └─────────────────────┘
```

**影响范围**: P5 页面、价格展示逻辑、客服对接

### 4. 增加进度条

**修改内容**:
- 在页面顶部添加 6 步进度指示器
- 显示当前步骤（如"步骤 3/6"）
- 已完成步骤显示绿色对勾

**修改原因**:
- 增强用户流程掌控感
- 明确当前位置和剩余步骤
- 提高完成率和用户满意度

**设计变更**:
```
新增全局进度条组件:

位置：导航栏下方，页面内容上方
高度：60px（含步骤文字）

状态:
  ○ 未完成（灰色 #CCCCCC）
  ● 进行中（蓝色 #1E88E5 + 数字）
  ✓ 已完成（绿色 #43A047 + 对勾）

各页面进度:
  P1: ● ○ ○ ○ ○ ○  步骤 1/6
  P2: ✓ ● ○ ○ ○ ○  步骤 2/6
  P3: ✓ ✓ ● ○ ○ ○  步骤 3/6
  P4: ✓ ✓ ✓ ● ○ ○  步骤 4/6
  P5: ✓ ✓ ✓ ✓ ● ○  步骤 5/6
  P6: ✓ ✓ ✓ ✓ ✓ ●  步骤 6/6
```

**影响范围**: 所有页面（P1-P6）、全局组件

### 6. AI 分析加载进度明确

**修改内容**:
- 使用进度条 + 百分比显示
- 添加分步提示（"识别中...分析中...生成方案..."）

**修改原因**:
- 减少用户等待焦虑
- 增强过程透明度
- 提升专业感

**设计变更**:
```
v2.0:
  正在分析您的照片...
  ✓ 识别漏水位置
  ✓ 分析严重程度
  ○ 估算面积
  ○ 匹配施工方案
  ━━━━━━━━━━━━ 75%
  预计还需 5 秒

v2.1（增强）:
  正在分析您的照片...
  
  ✓ 识别漏水位置      ← 绿色对勾 + 文字#333333
  ✓ 分析严重程度      ← 绿色对勾 + 文字#333333
  ● 估算面积          ← 蓝色图标 + 文字加粗#1E88E5
  ○ 生成施工方案      ← 灰色圆圈 + 文字#999999
  
  ━━━━━━━━━━━━━━━━ 75%  ← 进度条加粗至 6px
     (蓝色填充 + 百分比数字 14px Bold)
  
  预计还需 5 秒
```

**影响范围**: P3 页面、AI 分析加载状态

### 7. P2 图片上传数量提示

**修改内容**:
- 显示"已上传 X/5 张"
- 达到 5 张时禁用上传按钮

**修改原因**:
- 明确上传进度
- 防止超额上传
- 增强用户掌控感

**设计变更**:
```
v2.0:
  已上传 3 张

v2.1:
  已上传 3/5 张 ✅  ← 绿色对勾（达到 5 张时显示）
  
达到 5 张时:
  - 上传区域禁用（opacity 0.5）
  - 图标变为灰色 #CCCCCC
  - 提示文字："已达上限，无法继续上传"
  - 数量提示变绿色 #43A047 + ✅ 图标
```

**影响范围**: P2 页面、图片上传逻辑

## 📝 交互逻辑变更

### P3 交互变更

```
v2.0:
1. 进入 P3 → AI 模式默认展开
2. 自动开始 AI 分析
3. 用户等待分析完成

v2.1:
```

```
v2.0:
1. 填写姓名、手机、地址
2. 选择上门时间
3. 勾选协议
4. 点击"确认下单"→ P6

v2.1:
1. 填写姓名、手机（2 个字段）
2. 勾选协议
3. 点击"下一步"→ P6
4. P6 填写地址
5. 点击"提交订单"
```

```
v2.0:
1. 上传图片
2. 显示"已上传 X 张"
3. 达到 5 张时，上传区域变灰

v2.1:
1. 上传图片
2. 显示"已上传 X/5 张"
3. 达到 5 张时:

   - 图标变灰色
   - 提示"已达上限，无法继续上传"
   - 数量提示变绿色 + ✅ 图标
```

## 🔧 开发注意事项

### 前端开发

1. **全局进度条组件**
   - 需要封装为独立组件
   - 接收当前步骤参数（1-6）
   - 自动计算已完成/进行中/未完成状态
   - 动画效果：平滑过渡 0.3s

2. **P3 默认状态**
   - 默认展开手动输入模式
   - AI 模式默认收起
   - 添加提示文案组件

3. **P5 表单简化**
   - 移除地址字段
   - 添加表单进度提示
   - 修改下一步按钮逻辑（跳转 P6 而非提交）

4. **P6 地址字段**
   - 新增地址输入框
   - 必填校验（长度>10 字符）
   - 支持历史地址选择（可选）

5. **图片上传增强**
   - 实时更新数量提示
   - 达到 5 张时禁用上传
   - 添加禁用状态样式

6. **AI 加载进度**
   - 进度条加粗至 6px
   - 添加百分比数字显示
   - 分步提示状态实时更新

### 后端开发

1. **价格计算**
   - 明确返回"估算价格范围"
   - 添加价格说明字段

2. **表单提交**
   - P5 提交时不校验地址
   - P6 提交时校验地址
   - 分两步提交逻辑

3. **客服对接**
   - "预约顾问"按钮对接客服系统
   - 支持直接发起在线咨询
