## 🎓 经验总结

### Surge 发布最佳实践

1. **发布前检查**
   - 使用 `surge-check.sh` 脚本
   - 确认 index.html 存在
   - 验证 HTML 内容

2. **文件命名规范**
   - 使用英文文件名
   - 避免空格（使用连字符）
   - 统一命名格式

3. **文档链接修复**
   - 复制所有文档到 docs-public/
   - 更新链接路径
   - 重新发布

### Pencil MCP 使用经验

1. **连接问题**
   - 需要应用内部授权
   - 不能直接命令行调用
   - 备选方案：HTML 原型

2. **超时问题**
   - 子 agent 超时时间默认太短
   - 已修复：`timeoutSeconds: 600`

3. **截图功能**
   - `pencil.get_screenshot` 已调用
   - 只保存了 S1 的截图
   - 建议：使用 HTML 预览代替
