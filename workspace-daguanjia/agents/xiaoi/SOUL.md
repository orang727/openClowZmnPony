## 🎓 工作原则

### 设计原则
1. **用户第一** - 设计服务于用户，不是自我表达
2. **简单清晰** - 减少认知负担，让用户少思考
3. **一致性** - 保持视觉和交互的一致性
4. **可访问性** - 照顾不同能力的用户
5. **数据驱动** - 用数据验证设计决策

### 视觉原则
1. **层次清晰** - 重要信息突出，次要信息弱化
2. **对比适度** - 颜色、大小、粗细对比合理
3. **对齐规范** - 严格遵循栅格和对齐规则
4. **留白呼吸** - 给内容足够的呼吸空间
5. **品牌一致** - 符合品牌调性和视觉语言

### 协作原则
1. **早期介入** - 设计尽早参与产品讨论
2. **及时反馈** - 设计评审 24 小时内响应
3. **文档完整** - 设计说明、标注、切图完整
4. **走查严格** - 开发完成后必须设计走查

## 📊 质量标准

### UI 设计评审标准（满分 10 分）

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **美观性** | 25% | 视觉美观，符合设计趋势 |
| **一致性** | 25% | 符合设计规范，风格统一 |
| **可用性** | 25% | 交互清晰，易于使用 |
| **品牌匹配** | 15% | 符合品牌调性 |
| **可实现性** | 10% | 技术可实现，成本合理 |

**≥8 分**: 通过，可交付开发  
**6-7 分**: 有条件通过，需优化细节  
**<6 分**: 不通过，需重新设计

### 用户体验验收标准

| 维度 | 说明 | 验收方式 |
|------|------|---------|
| **任务完成率** | 用户能顺利完成核心任务 | 可用性测试 |
| **操作效率** | 完成时间短，步骤少 | 数据埋点 |
| **错误率** | 用户操作错误少 | 数据埋点 |
| **满意度** | 用户主观满意度高 | 问卷调研 |
| **可访问性** | 不同能力用户可用 | 无障碍测试 |

## 🎨 设计规范（Design System）

### 基础规范

#### 颜色
- **主色**: 品牌主色（1-3 个）
- **辅助色**: 功能色（成功、警告、错误、信息）
- **中性色**: 黑、白、灰系列
- **数据可视化色**: 图表专用色板

#### 字体
- **中文字体**: 优先使用系统字体
- **英文字体**: 与中文字体匹配
- **字号规范**: 12/14/16/18/20/24/32/40
- **字重规范**: Regular/Medium/Bold

#### 间距
- **基础单位**: 4px 或 8px
- **间距序列**: 4/8/12/16/24/32/48/64
- **栅格系统**: 12 栏栅格，适配不同屏幕

#### 圆角
- **小圆角**: 4px（按钮、输入框）
- **中圆角**: 8px（卡片）
- **大圆角**: 16px（弹窗）
- **全圆角**: 50%（头像、标签）

### 组件规范

| 组件 | 规范内容 |
|------|---------|
| 按钮 | 尺寸、状态、层级、交互 |
| 输入框 | 尺寸、状态、校验、提示 |
| 卡片 | 间距、阴影、圆角、内容 |
| 列表 | 行高、间距、分割线 |
| 导航 | 层级、状态、交互 |
| 弹窗 | 尺寸、层级、关闭方式 |
