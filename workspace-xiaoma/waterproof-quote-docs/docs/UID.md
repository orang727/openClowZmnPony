## 二、页面详细设计

### 2.1 顶部品牌区

| 元素 | 内容 |
|------|------|
| 角标 | 智能防水 · 报价助手（绿色圆点 + 蓝色标签） |
| 主标题 | 防水补漏 · 一键智能估价 |
| 副标题 | 拍几张现场照片 + 填写基础信息，系统将基于历史案例与经验规则，为你智能评估漏水严重程度并推荐施工方案与价格区间。 |

**设计规范**：
- 主标题：20px，加粗，深色
- 副标题：13px，灰色
- 角标：11px，蓝色背景，圆角

### 2.2 Step 1：漏点位置与基本情况

#### 卡片样式
- 背景：白色
- 圆角：14px
- 阴影：0 8px 24px rgba(15, 23, 42, 0.06)
- 间距：12px 底部外边距

#### 字段说明

| 字段 | 组件 | 必填 | 说明 |
|------|------|------|------|
| 漏点位置 | van-field + van-picker | 是 | 下拉选择：卫生间/厨房/阳台/屋顶/外墙/飘窗/其他 |
| 楼层 | van-field + van-picker | 否 | 下拉选择：低层/中层/高层 |
| 房龄 | van-field + van-picker | 否 | 下拉选择：0-5年/5-10年/10年以上 |
| 问题描述 | van-field (textarea) | 否 | 最大200字，显示字数统计 |

**交互设计**：
- 点击 van-field 弹出 van-picker（底部弹起）
- 选择后自动回显，picker 自动关闭

### 2.3 Step 2：上传现场图片

#### 卡片样式
- 同 Step 1 卡片样式

#### 上传区域
- 点击区域：虚线边框 + 渐变背景（浅蓝到浅紫）
- 图标：📷 相机图标
- 文案：
  - 主文案：点击拍照或从相册选择
  - 副文案：支持最多 5 张，单张不超过 8MB

#### 底部提示
- 建议拍摄 1 张整体环境 + 1~3 张近距离细节
- 请避开隐私信息，如家庭成员正脸、证件等

**交互设计**：
- 点击打开相册/相机
- 上传后显示缩略图
- 支持删除和重新上传
- 最多 5 张

### 2.4 Step 3：智能估价结果

#### 卡片样式
- 同上述卡片样式
- 顶部边框：3px 蓝色（区分结果区域）

#### 严重程度展示

| 严重程度 | 背景色 | 标签色 |
|----------|--------|--------|
| 轻微 (light) | #ecfdf5 (浅绿) | #16a34a (绿色) |
| 普通 (normal) | #f3f4ff (浅蓝) | #1d4ed8 (蓝色) |
| 严重 (severe) | #fef2f2 (浅红) | #dc2626 (红色) |

**布局**：
- 左侧：严重程度标签 + 分析说明
- 右侧：初步评估角标

#### 方案卡片列表

**单个方案卡片**：
- 边框：1px solid #e5e7eb
- 圆角：12px
- 内边距：10px
- 选中状态：边框变蓝色 + 蓝色阴影

**方案内容**：
- 方案标题 + 标签（标签为蓝色小圆角）
- 方案描述
- 元信息行：预计工期 | 参考价格区间
- 底部：价格说明 + 单选按钮

**价格样式**：
- 价格数字：13px，加粗
- 价格颜色：#b45309 (橙色)

### 2.5 兜底状态

当用户未填写信息时：
- 显示工具图标 🛠️
- 文案：还没有估价结果
- 副文案：填写上面的信息并上传现场图片，点击下方「智能估价」即可获得方案与价格区间。

## 三、底部操作条

### 3.1 布局

- 固定在页面底部
- 背景：半透明渐变
- 阴影：顶部投影

### 3.2 内容

| 左侧 | 右侧 |
|------|------|
| 状态标签（当前状态：待填写信息/正在智能分析中.../已生成估价结果） | 操作按钮 |

### 3.3 按钮状态

| 场景 | 按钮文案 | 按钮类型 | 状态 |
|------|----------|----------|------|
| 未估价 | 智能估价 | primary (蓝色) | 漏点位置+图片必填，否则禁用 |
| 估价中 | 智能估价 | primary | loading 状态 |
| 已估价 | 生成工单 | warning (橙色) | 选择方案后启用，否则禁用 |
| 提交中 | 生成工单 | warning | loading 状态 |

## 四、交互流程设计

### 4.1 完整交互流程

```
1. 用户进入页面
   │
   ▼
2. 填写表单（漏点位置必填）
   │
   ▼
3. 上传图片（至少1张）
   │
   ▼
4. 点击「智能估价」按钮
   │
   ▼
5. 按钮进入 loading 状态
   │
   ▼
6. 后端调用 AI 分析图片
   │
   ▼
7. 返回严重程度 + 方案列表
   │
   ▼
8. 页面展示结果区域
   │
   ▼
9. 用户选择方案卡片
   │
   ▼
10. 点击「生成工单」按钮
    │
    ▼
11. 按钮进入 loading 状态
    │
    ▼
12. 后端创建工单
    │
    ▼
13. 弹出成功对话框
    │
    ▼
14. 用户确认 → 回调小程序
```

### 4.2 状态管理

| 状态 | 说明 |
|------|------|
| `form` | 表单数据对象 |
| `loading` | 智能估价请求中 |
| `confirming` | 生成工单请求中 |
| `hasResult` | 是否已有估价结果 |
| `selectedPlanId` | 用户选中的方案ID |
| `result` | 估价结果数据 |

### 4.3 表单校验规则

| 字段 | 校验规则 |
|------|----------|
| 漏点位置 | 必须选择 |
| 图片 | 至少上传 1 张 |
| 楼层/房龄/描述 | 可选 |

## 六、异常处理

### 6.1 网络错误

- 智能估价失败：显示 Toast 提示「网络错误，请稍后重试」
- 生成工单失败：显示 Toast 提示「创建工单失败，请稍后重试」

### 6.2 校验失败

- 未选择漏点位置：Toast 提示「请先选择漏点位置」
- 未上传图片：Toast 提示「请先上传至少一张现场图片」

### 6.3 AI 分析失败

- 返回默认严重程度（普通）
- 显示默认方案列表
