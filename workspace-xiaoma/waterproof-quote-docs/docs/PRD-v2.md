## 一、产品概述

### 1.1 产品定位

| 项目 | 说明 |
|------|------|
| **产品名称** | 防水智能报价工具 |
| **产品形态** | H5 页面（嵌入官网品牌站） |
| **战略定位** | 获客子模块（前端引流工具，非独立盈利产品） |
| **核心价值** | 透明报价建立信任，降低获客成本 |
| **目标用户** | 有防水补漏需求的家庭业主、物业/业委会 |
| **上线渠道** | 官网品牌站（H5 内嵌） |

### 1.2 战略背景

**市场规模**：家庭防水市场约 200-300 亿，存量房维修爆发期

**行业痛点**：
- "注胶按斤收费"模式导致大量欺诈投诉
- 平台 30-40% 抽佣逼迫工程师虚增材料费
- 行业声誉极差，用户信任成本极高

**产品价值**：用标准化 + 专业化填补头部和尾部之间的巨大中间市场，通过透明报价建立用户信任。

### 1.3 核心指标

| 指标 | 目标值 | 说明 |
|------|-------|------|
| 使用量 | 月均 10000+ 次 | 引流能力 |
| 留资率 | 15-25% | 报价→留资转化 |
| 上门转化率 | 50%+ | 留资→上门 |
| 签约转化率 | 30%+ | 上门→施工 |
| 获客成本 | <100 元/单 | 对比 SEM 显著降低 |

## 二、用户角色与场景

### 2.1 目标用户

| 优先级 | 用户类型 | 场景 | 获客价值 |
|-------|---------|------|---------|
| **P0** | 家庭业主 | 卫生间/厨房/墙面渗漏 | 高（C 端主力） |
| **P1** | 物业/业委会 | 公共区域维修 | 极高（B 端） |
| **P2** | 小微企业 | 办公室/商铺 | 中 |

### 2.2 用户流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  进入 H5    │ ──► │ 选择问题类型 │ ──► │  填写信息   │ ──► │ 获取报价    │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                                                                     │
                                                                     ▼
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  客服跟进   │ ◄── │ 提交留资    │ ◄── │ 查看详细方案 │ ◄── │ 基础报价展示 │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

### 2.3 6 步流程设计

| 步骤 | 内容 | 必填 | 说明 |
|------|------|------|------|
| Step 1 | 问题类型选择 | 是 | 卫生间/厨房/阳台/屋顶/外墙/飘窗/其他 |
| Step 2 | 面积输入 | 是 | 平方米或区间选择 |
| Step 3 | 渗漏程度评估 | 是 | 轻微/普通/严重（用户自评 +AI 辅助） |
| Step 4 | 基础报价展示 | - | **直接显示**价格区间（建立信任） |
| Step 5 | 留资获取详细方案 | 是 | 手机号 + 称呼 + 小区（可选） |
| Step 6 | 客服跟进 + 上门检测 | - | 销售跟进，安排上门 |

## 三、功能需求

### 3.1 功能清单

| 序号 | 功能模块 | 功能描述 | 优先级 | 开发方式 |
|------|----------|----------|--------|---------|
| 1 | 问题类型选择 | 用户选择漏水位置 | P0 | AI 生成 |
| 2 | 面积输入 | 输入或选择面积区间 | P0 | AI 生成 |
| 3 | 渗漏程度评估 | 用户自评 +AI 辅助判断 | P0 | AI 生成 |
| 4 | 基础报价展示 | 直接显示价格区间 | P0 | AI 生成 |
| 5 | 详细方案展示 | 留资后查看完整方案 | P0 | AI 生成 |
| 6 | 留资表单 | 手机号 + 称呼 + 小区 | P0 | AI 生成 |
| 7 | 报价算法 | 根据位置/面积/程度计算价格 | P0 | AI 生成 |
| 8 | 留资数据管理 | 后台查看留资数据 | P1 | AI 生成 |
| 9 | 数据埋点 | 用户行为追踪 | P1 | AI 生成 |
| 10 | 工单推送 | 推送到工单系统 | P2 | 线下员工 |

### 3.2 功能详细说明

#### 3.2.1 问题类型选择（P0）

- **组件**：卡片选择器
- **选项**：
  - 卫生间（默认推荐）
  - 厨房
  - 阳台
  - 屋顶
  - 外墙
  - 飘窗
  - 其他
- **交互**：点击选中，高亮显示

#### 3.2.2 面积输入（P0）

- **输入方式**：数字输入框 或 区间选择
- **区间选项**：
  - <5㎡
  - 5-10㎡
  - 10-20㎡
  - 20-50㎡
  - >50㎡
- **校验**：必填，最大值限制 1000㎡

#### 3.2.3 渗漏程度评估（P0）

| 程度 | 用户描述 | 价格系数 |
|------|---------|---------|
| **轻微** | 表面潮湿，无明显渗水 | 1.0 |
| **普通** | 有明显水渍，偶尔滴水 | 1.5 |
| **严重** | 持续渗水，影响使用 | 2.0 |

- **辅助判断**：提供示例图片帮助用户判断
- **AI 辅助**：后续版本支持上传图片 AI 识别

#### 3.2.4 基础报价展示（P0）

- **展示形式**：￥800 - ￥1200
- **说明文案**： "以上为预估价格区间，实际价格以上门检测为准"
- **展示时机**：用户完成 Step 1-3 后**直接显示**（无需留资）
- **战略意义**：建立信任，提高留资率

#### 3.2.5 详细方案展示（P0）

- **触发条件**：用户提交留资信息后
- **方案内容**：
  - 推荐施工方案（经济型/标准型/加强型）
  - 施工流程说明
  - 预计工期
  - 材料说明
  - 质保期限
  - 详细价格

#### 3.2.6 留资表单（P0）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 手机号 | 数字输入 | 是 | 11 位手机号，验证码校验（可选） |
| 称呼 | 文本输入 | 是 | 先生/女士 或 自定义 |
| 小区名称 | 文本输入 | 否 | 方便师傅定位 |
| 预约时间 | 选择器 | 否 | 方便的时间段 |

#### 3.2.7 报价算法（P0）

**计算公式**：
```
基础价格 = 单价 × 面积
最终价格 = 基础价格 × 位置系数 × 渗漏系数
```

**系数表**：

| 位置 | 系数 | 说明 |
|------|------|------|
| 卫生间 | 1.2 | 施工复杂，需要拆改 |
| 厨房 | 1.1 | 中等复杂度 |
| 阳台 | 1.0 | 标准施工 |
| 屋顶 | 1.3 | 高空作业，难度大 |
| 外墙 | 1.5 | 高空作业 + 特殊材料 |
| 飘窗 | 1.1 | 中等复杂度 |

| 渗漏程度 | 系数 | 说明 |
|---------|------|------|
| 轻微 | 1.0 | 表面防水即可 |
| 普通 | 1.5 | 需要局部开挖 |
| 严重 | 2.0 | 全面重做防水层 |

**单价参考**：
- 卫生间/厨房：80-120 元/㎡
- 阳台/屋顶：60-100 元/㎡
- 外墙：100-150 元/㎡

#### 3.2.8 留资数据管理（P1）

- **功能**：后台查看留资用户信息
- **字段**：手机号、称呼、小区、问题类型、面积、程度、报价、提交时间
- **导出**：支持 Excel 导出
- **推送**：自动推送给客服/销售

#### 3.2.9 数据埋点（P1）

| 事件 | 触发时机 | 数据内容 |
|------|---------|---------|
| page_view | 页面加载 | 用户 ID、来源渠道 |
| step_complete | 完成每个步骤 | 步骤编号、耗时 |
| quote_show | 报价展示 | 报价金额、位置、面积 |
| form_submit | 留资提交 | 手机号、完整表单数据 |
| exit | 用户离开 | 离开时的步骤、停留时长 |

## 四、非功能性需求

### 4.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 页面加载时间 | < 3 秒 | 首屏加载 |
| 报价计算响应 | < 1 秒 | 用户操作后即时显示 |
| 表单提交响应 | < 2 秒 | 提交后即时反馈 |
| 并发支持 | 1000+ QPS | 支持高并发访问 |

### 4.2 兼容性

- **浏览器**：Chrome、Safari、微信内置浏览器、抖音内置浏览器
- **系统**：iOS 12+、Android 8+
- **屏幕**：支持主流手机屏幕尺寸（320px - 480px 宽度）

### 4.3 安全要求

- **数据传输**：HTTPS 加密传输
- **敏感信息**：手机号等敏感信息加密存储
- **防刷机制**：IP 限流、表单提交频率限制
- **隐私合规**：符合隐私政策要求

### 4.4 可用性

- **可用性**：99.9%
- **容错**：关键操作有确认提示
- **异常处理**：网络异常、服务器错误有友好提示

## 七、风险与应对

| 风险 | 等级 | 影响 | 应对措施 |
|------|------|------|---------|
| 报价不准被投诉 | 高 | 用户投诉、品牌受损 | 设置价格浮动范围，明确告知"估算价"，建立快速客诉响应机制 |
| 开发延期 | 高 | 无法按时上线 | 优先保证核心功能（报价 + 留资），非核心功能延后，每周检查进度 |
| 师傅调研配合度低 | 中 | 算法准确性受影响 | 提供调研激励（红包/积分），简化调研流程，通过运营强制推动 |
| 留资率不达标 | 中 | 获客效果差 | 优化留资引导文案，提供"预约上门"等增值服务，A/B 测试优化转化率 |
| 竞品模仿 | 低 | 竞争优势减弱 | 快速迭代建立壁垒，强化品牌和用户心智，持续优化算法准确度 |
